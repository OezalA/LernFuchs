A small web application to help a 5th-grade Gymnasium student improve her
**vocabulary (Wortschatz)** and **reading comprehension (LeseverstГӨndnis)** вҖ” in
**German** (her native/school language) and in **English, Spanish and French as
foreign languages**. A **"Du Гјbst: вҖҰ"** switch in the header (with a *"Wechseln zu"*
menu) picks which language to practise; everything else (games, progress, streak)
stays the same.

Learning content (vocabulary lists, reading texts and questions) is generated by
Google **Gemini** (free tier) and stored locally, so the app can be used offline
afterwards without repeated API calls.

> **Languages:** The **user interface stays German** (the child is a German speaker).
> The *learning content* adapts to how new each language is:
> - **German** вҖ” full immersion: words, example sentences and meaning explanations
>   are all in child-friendly German.
> - **English (foreign language, A1вҖ“A2)** вҖ” deliberately easier: short English texts
>   and simple questions, but each new word is **explained in German**. An optional
>   word-learning step (flash-cards with pronunciation) is offered *before* reading.
> - **Spanish & French (absolute beginner, pre-A1)** вҖ” the child starts from zero, so
>   these are gentler still: very short texts, the **comprehension questions are in
>   German**, and **every word of the text** goes into the word-learning step.
>
> In every foreign language the reader is **sentence-by-sentence**: tapping a sentence
> reads it aloud and shows its **German translation**. All this help text is generated
> once (at content-creation time) and stored, so no AI is called while the child reads.
>
> This documentation is in English.

### Domain model

- **VocabularyWord** вҖ” a word with article (der/die/das), plural, word type, a
  child-friendly German definition (for foreign words: the German translation),
  example sentence, synonyms/antonyms, difficulty, topic and a **`Language`**
  (Deutsch / Englisch / Spanisch / FranzГ¶sisch).
- **VocabularyProgress** вҖ” spaced-repetition state per word using the **Leitner system**
  (boxes 0вҖ“5; box 5 = *learned*).
- **ReadingPassage** вҖ” a reading text with comprehension questions and a **`Language`**.
  For foreign languages it also stores a **sentence-by-sentence German translation**,
  and for the absolute-beginner languages a **full word glossary** of the text.
- **ComprehensionQuestion** вҖ” multiple-choice or open-ended question with the correct
  answer and an explanation.

## Configuration

The Gemini API key must **never** be committed. Use .NET User Secrets during development:

```bash
cd backend/LernFuchs.Api
dotnet user-secrets init
dotnet user-secrets set "Gemini:ApiKey" "YOUR_API_KEY_HERE"
```

Alternatively, set it via an environment variable:

```bash
# PowerShell
$env:Gemini__ApiKey = "YOUR_API_KEY_HERE"
```

The model can be changed in `appsettings.json` under `Gemini:Model`.

## Running the frontend

In a second terminal (keep the backend running):

```bash
cd frontend
npm install     # first time only
npm start
```

Then open **http://localhost:4200**. The dev server proxies all `/api` calls to the
backend on port 5219 (see `proxy.conf.json`), so no CORS setup is needed.

The UI (in German) has a start page with a **progress dashboard**, a **"Du Гјbst: вҖҰ"
language switch** (Deutsch / Englisch / Spanisch / FranzГ¶sisch, with a *"Wechseln zu"*
menu) in the header, and three areas:

- **Wortschatz** вҖ” the words the child has met, grouped by topic, with **read-aloud**
  (рҹ”Ҡ). Two practice modes:
  - **рҹҺҙ Гңben** вҖ” flip-cards; marking **"Gewusst"** moves a word to *learned*.
  - **вң… Test** вҖ” multiple choice in random order; getting a word right **4 times in
    a row** marks it *learned*.
- **LeseverstГӨndnis** вҖ” read a short text (with optional **read-aloud**), filter by
  topic, answer the questions and get instant per-question feedback and a score.
  The **difficult words are extracted into the Wortschatz**, and the newest texts are
  marked **"neu"**. In a **foreign language** the reader works **sentence by sentence**
  (tap a sentence to hear it and see its German translation) and offers an optional
  **word-learning step** (flash-cards, pronunciation вҶ’ German meaning) before reading вҖ”
  for **Spanish/French** that step covers *every* word of the (very short) text.
- **Spiele** вҖ” practice games that reuse the vocabulary and award XP: **Quiz**,
  **der/die/das** (article), **Diktat** (listen and type), **Memory**, and
  **Verben** (conjugation). The German-only games (der/die/das, Verben) are hidden
  in the foreign-language modes.

Content is scoped to the selected language (reading, vocabulary, games and the
dashboard stats); XP, level and the daily streak stay shared across both languages.

### Gamification & progress

- **XP, levels and a daily streak** (рҹ”Ҙ) shown in the top bar; correct answers earn XP.
- A **daily-goal ring** and **achievement badges** on the start page.
- **Charts**: XP per day over the last week and the Leitner box distribution
  (`GET /api/stats/progress`).
- Confetti and Web-Audio **sound effects** with a global mute toggle.

The start page also shows live stats: words collected, mastered, due today, success
rate and number of reading texts (`GET /api/stats`).

## API endpoints

### Vocabulary вҖ” `/api/vocabulary`

| Method | Route                       | Description                                        |
| ------ | --------------------------- | -------------------------------------------------- |
| GET    | `/api/vocabulary`           | List words (filter by `topic`, `difficulty`, `language`) |
| GET    | `/api/vocabulary/{id}`      | Get one word                                       |
| GET    | `/api/vocabulary/due`       | Words due for review (Leitner system)              |
| POST   | `/api/vocabulary/generate`  | Generate & save new words via Gemini               |
| POST   | `/api/vocabulary/{id}/review` | Record a review result and update progress       |
| POST   | `/api/vocabulary/{id}/learned` | Mark a word as learned (top Leitner box)         |
| DELETE | `/api/vocabulary/{id}`      | Delete a word                                       |

### Reading вҖ” `/api/reading`

| Method | Route                     | Description                                          |
| ------ | ------------------------- | ---------------------------------------------------- |
| GET    | `/api/reading`            | List passages (filter by `topic`, `difficulty`, `language`) |
| GET    | `/api/reading/{id}`       | Get a passage: questions (answers hidden), difficult words, sentence translations |
| POST   | `/api/reading/generate`   | Generate & save a new passage + questions via Gemini |
| POST   | `/api/reading/{id}/check` | Submit answers and get per-question feedback         |
| DELETE | `/api/reading/{id}`       | Delete a passage                                      |

### Example: generate vocabulary

```bash
curl -X POST https://localhost:<port>/api/vocabulary/generate \
  -H "Content-Type: application/json" \
  -d '{ "topic": "Schule", "difficulty": "Mittel", "count": 10 }'
```

## Testing

Backend tests use **xUnit** with an in-memory SQLite database and a fake content
service (no real Gemini calls), covering the Leitner review logic, answer grading,
difficult-word deduplication and that correct answers are never leaked by the API.

```bash
cd backend
dotnet test
```

Pushes to `develop` and pull requests to `main` / `develop` run build + tests via
GitHub Actions ([`ci.yml`](.github/workflows/ci.yml)). Deployments to `main` run the
same tests first and **only deploy if they pass** (see below).

## Deployment

The app is deployed as a **single package** to **Azure App Service** (Linux, free F1
tier). During `dotnet publish` the Angular frontend is built and copied into the API's
`wwwroot`, so one ASP.NET Core process serves both the static SPA and the `/api`
endpoints (with an SPA fallback route). SQLite lives on a persistent path
(`/home/data`) so generated content survives restarts.

Deployment is **automated with GitHub Actions** ([`deploy.yml`](.github/workflows/deploy.yml)):
every push to `main` first **builds and runs the tests**, and only if they pass builds
the publish bundle and ships it to Azure. Authentication uses **passwordless OpenID
Connect (OIDC)** вҖ” no publish profile or password is stored.

In production, on-demand content generation is disabled and a background service
(`DailyContentService`) generates a few fresh reading texts per day automatically
(German, English, Spanish and French) вҖ” so there is nothing to run locally and no
computer needs to stay on.
